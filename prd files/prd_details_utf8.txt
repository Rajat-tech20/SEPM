1. PROJECT OVERVIEW
This project involves building a multi-page static educational website on the topic of Software Engineering,
intended for engineering students. The website will be structured across five dedicated pages, each covering
a distinct academic topic. Page 1 (Home) content has been provided by the client. Content for Pages 2–5 is
awaiting client delivery and will be plugged in upon receipt. The developer's responsibility is to deliver a
complete, styled, and navigable website scaffold — fully functional with placeholder-ready content zones.

7. ASSUMPTIONS & DEPENDENCIES

Client will provide textual content for Pages 2–5 before the Phase 3 deadline. Delays will push final delivery.
No backend, database, or server is required. This is a purely static website.
Google Fonts may be used for typography — considered a safe external dependency for a student project.
UML Diagrams page may receive image assets from the client; image slots will be pre-built with defined
dimensions.
Test Cases page may require a table layout — developer will build a reusable table component ready for data.
The website does not need to be deployed online; it must work locally via file:// protocol.
