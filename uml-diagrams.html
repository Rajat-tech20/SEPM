UML Diagrams.

Unified Modeling Language for system architecture.

STATIC UML Diagram

UML visualizations to describe both structural and behavioral aspects of a system.

AIM:

Objective: To visualize the "backbone" of a system its physical or logical structure—without regard for time or dynamic behaviour.

Class Diagram

A UML class diagram shows the structure of a system by displaying its classes, their attributes, methods, and the relationships between them.

Component Diagram

A component diagram is a high-level structural diagram in Unified Modeling Language (UML) that visualizes how a system is divided into modular, replaceable parts and how those parts interact.

Deployment Diagram

A UML deployment diagram visualizes the physical, runtime architecture of a system, mapping software components (artifacts) to hardware infrastructure (nodes).

Practical No. 6

Aim: To design dynamic UML Diagrams.

AIM:

Objective: To model and visualize the dynamic behaviour of a system over time.

Use Case Diagram

A Use Case Diagram is used to represent the functional requirements of a system from the user's perspective. It shows how external actors interact with the system.

(Note: for notations refer class notes draw on blank page)

Activity Diagram

An Activity Diagram is used to model the workflow or process flow of a system. It visually represents the sequence of activities and actions, including decision points and parallel processes.
