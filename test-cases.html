Test Cases.

Validating functionality, safety, and performance.

PRACTICAL NO. 7

Aim: To design Test cases for the assigned application.

Objective:

To design and validate test cases for a system to ensure correct functionality, input validation, and security.

Theory: Test Case Theory (Short & Clear)

A test case is a set of conditions or steps used to verify that a software application works correctly. It helps testers check whether a feature (like login) behaves as expected.

🔹 Key Components of a Test Case

  • Test Case ID – Unique identifier
  • Description – What is being tested
  • Test Steps – Actions to perform
  • Test Data – Input values
  • Expected Result – Expected outcome
  • Actual Result – Actual outcome after execution
  • Status – Pass/Fail

🔹 Types of Test Cases

  • Positive Test Cases – Check valid inputs (correct login)
  • Negative Test Cases – Check invalid inputs (wrong password)

🔹 Importance of Test Cases

  • Ensures software quality and reliability
  • Helps identify bugs and errors
  • Provides clear testing documentation
  • Improves user experience
